## Описание скрипта: «Универсальный скрипт для работы с видео»

Данный PowerShell-скрипт предназначен для автоматизации двух основных задач, связанных с обработкой видеофайлов:

1. **Классификация (сортировка) видео** – анализ технических параметров видеофайлов (разрешение, соотношение сторон, частота кадров, битрейт, наличие аудиодорожки) и последующее копирование/перемещение/создание ярлыков в структурированные папки.
2. **Замена несжатых видео сжатыми** – поиск соответствий между файлами в двух папках по имени (без учёта расширения) и замена исходных несжатых файлов сжатыми версиями.

Скрипт имеет интерактивное меню, запрашивает пути, параметры фильтрации и стратегию обработки дубликатов.

### Режим 1. Классификация видео

#### Назначение
Упорядочить видеофайлы по папкам в зависимости от их характеристик. Например, все файлы с соотношением 16:9, разрешением 1080p и частотой 25 fps попадут в папку `16-9_1080p_25fps`.

#### Возможности
- **Анализ** видео с помощью `ffprobe` (из пакета FFmpeg). Считываются:
  - битрейт;
  - ширина и высота;
  - частота кадров (FPS);
  - наличие аудиопотока.
- **Фильтрация** по:
  - минимальному битрейту (кбит/с);
  - списку разрешений (2160p, 1080p, 720p и т.д.);
  - списку соотношений сторон (16x9, 4x3, 21x9, 9x16 и др.);
  - списку FPS (24, 25, 30, 50, 60 и т.д.);
  - наличию/отсутствию звука.
- **Стратегия именования папок** (выбирается пользователем):
  - **Полная сортировка**: `{соотношение}_{разрешение}_{FPS}` (например `16-9_1080p_30fps`).
  - **Только по FPS**: `{FPS}` (например `25fps`).
- **Действие с файлами**:
  - Копирование (исходники остаются);
  - Перемещение (исходники удаляются);
  - Создание ярлыков `.lnk`.
- **Обработка дубликатов** (когда в одной целевой папке уже есть файл с таким же именем):
  - автоматическое переименование (добавление `_1`, `_2` …);
  - пропуск;
  - запрос действия для каждого конфликта.
- **Логирование** в консоль с цветовой разметкой, прогресс-бар.
- **Итоговая статистика**: количество обработанных/пропущенных/ошибочных файлов, общий объём.

#### Требования
- Установленный **FFmpeg** (требуется `ffprobe.exe`). Путь к нему запрашивается при запуске. (https://ffmpeg.org/download.html)
- PowerShell 5.1 и выше (возможна работа в Windows PowerShell и PowerShell 7).

### Режим 2. Замена несжатых видео сжатыми

#### Назначение
Заменить «тяжёлые» несжатые видеофайлы их сжатыми версиями, сохранив исходные имена и расположение. Часто используется после перекодирования.

#### Логика работы
1. Пользователь указывает **папку со сжатыми видео** (источник) и **папку с несжатыми видео** (цель).
2. Скрипт сканирует папку источника на наличие видеофайлов (расширения из списка `$videoExtensions`).
3. Для каждого сжатого файла извлекается **имя без расширения** и ищется любой файл с таким же базовым именем в целевой папке (рекурсивно, по первому совпадению).
4. Если соответствие найдено, сжатый файл **копируется** или **перемещается** поверх целевого (несжатого) файла.
5. Результат записывается в лог-файл `Replacement.log` рядом со скриптом.

#### Дополнительно
- Перед началом запрашивается подтверждение действия.
- Выводится подробная статистика: сколько файлов успешно заменено, сколько не найдено, сколько ошибок.
- В консоли и в логе фиксируются пути и статус каждой операции.

#### Особенности
- При **копировании** сжатый файл остаётся в исходной папке, а несжатый перезаписывается.
- При **перемещении** сжатый файл удаляется из папки источника после успешной замены.
- Расширение файлов может отличаться (например, `video.mkv` заменяет `video.avi`).

### Общие настройки и зависимости
- **Поддерживаемые форматы**: `mp4, mkv, avi, mov, wmv, flv, webm, m4v, mpg, mpeg, ts, m2ts, 3gp, hevc, h265`.
- Для работы режима классификации требуется `ffprobe.exe`. Режим замены не зависит от FFmpeg.
- Скрипт не изменяет метаданные видео, работает только на уровне файловой системы.

### Пример запуска
1. Сохранить скрипт как `WorkWithVideo.ps1`.
2. При необходимости разрешить выполнение сценариев:  
   `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
3. Запустить:  
   `.\WorkWithVideo.ps1`
4. Выбрать режим (1 или 2) и следовать подсказкам.

Скрипт написан с учётом удобства пользователя: цветной вывод, запросы с пояснениями, защита от случайной перезаписи, прогресс-бары при длительных операциях.
